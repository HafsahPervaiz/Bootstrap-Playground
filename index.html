Unit I — Scaffolding

Introduction to Bootstrap

What is Bootstrap, why use it, and how does it fit into modern web development?

📚 Understanding Bootstrap

Bootstrap is the world's most popular open-source CSS framework, originally created by Mark Otto and Jacob Thornton at Twitter in 2011. It provides pre-written CSS and JavaScript to quickly build responsive, mobile-first websites.

Why use Bootstrap? It gives you ready-to-use components (buttons, navbars, cards, forms) so you can focus on what your site looks like rather than coding basic styling from zero.

Key advantages:

  • Responsive Design Built-In: Components automatically adapt to different screen sizes using a mobile-first approach.
  • Consistent Cross-Browser Support: Your site looks the same everywhere.
  • Rapid Prototyping: Build professional interfaces in minutes.
  • Huge Community: Extensive documentation and thousands of themes.
  • Customizable: Override defaults with your own CSS or Sass variables.
💡 Good to know: Bootstrap follows a mobile-first philosophy — styles are designed for small screens first, then enhanced for larger screens using media queries.
📖 Key Concepts
  • CSS Framework — Pre-written CSS + JS for rapid development
  • Mobile-First — Design for small screens first, scale up
  • Responsive Design — Layouts that adapt to any screen
  • Utility Classes — Single-purpose classes like .text-center, .mt-3
  • Components — Pre-built UI elements (navbar, cards, modals)
  • Grid System — 12-column layout system

Unit I — Scaffolding

File Structure & HTML Template

How Bootstrap is organized and how to set up a basic Bootstrap project.

📚 Understanding Bootstrap's Structure

Key files: bootstrap.min.css (all styles), bootstrap.bundle.min.js (JS + Popper.js for tooltips/dropdowns).

Using a CDN is the easiest way — just add <link> and <script> tags.

Every Bootstrap page needs:

  1. HTML5 doctype: <!DOCTYPE html>
  2. Responsive viewport meta tag
  3. Bootstrap CSS link in <head>
  4. Bootstrap JS bundle before </body>
💡 Always include the viewport meta tag — without it, responsive design fails on mobile.
📖 Bootstrap File Organization
  • bootstrap/css/ — Compiled CSS files
  • bootstrap/js/ — Compiled JS files
  • bootstrap.min.css — Use in production
  • bootstrap.bundle.min.js — JS + Popper combined
  • <!DOCTYPE html> — Required doctype
  • viewport meta tag — Required for responsive
👀 Starter Template
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport"
    content="width=device-width,
    initial-scale=1.0">
  <link href="bootstrap.min.css"
    rel="stylesheet">
  <title>My Page</title>
</head>
<body>
  <h1>Hello World!</h1>
  <script src="bootstrap.bundle.min.js">
  </script>
</body>
</html>
Unit I — Scaffolding

Containers & Responsive Design

The building blocks of Bootstrap layouts.

📚 Understanding Containers

Containers are the most fundamental layout element. Three types:

  • .container (Fixed): Has a fixed max-width at each breakpoint. Content is centered.
  • .container-fluid (Fluid): Always 100% width, edge to edge.
  • .container-{breakpoint} (Responsive): Fluid below breakpoint, fixed above.

Breakpoints: xs(<576px), sm(≥576px), md(≥768px), lg(≥992px), xl(≥1200px), xxl(≥1400px)

💡 The .container class is almost always your first wrapper. Put rows and columns inside it.
📖 Key Classes
  • .container — Fixed-width, centered
  • .container-fluid — 100% width always
  • .container-sm — Fluid until ≥576px
  • .container-md — Fluid until ≥768px
  • .container-lg — Fluid until ≥992px
  • .container-xl — Fluid until ≥1200px

Unit II — Grid System

Bootstrap Grid System

The 12-column grid is Bootstrap's most powerful feature.

📚 Understanding the Grid

The grid uses containers, rows, and columns. It divides any row into 12 equal columns. Two equal columns? .col-6 + .col-6 (6+6=12). Sidebar layout? .col-8 + .col-4 (8+4=12).

Responsive columns: .col-12 .col-md-6 .col-lg-4 = full width on phones, half width on tablets, one-third on desktops.

Gutters: Use .g-0 to .g-5 on the .row to control spacing between columns.

💡 Always follow: .container → .row → .col-*. Columns must be direct children of rows.
📖 Key Classes
  • .row — Creates a horizontal group of columns
  • .col-6 — Takes 6 of 12 columns (50%)
  • .col-md-4 — 4 columns on md screens and up
  • .col-auto — Sizes to its content
  • .g-3 — Gutter spacing on rows
  • Breakpoints: sm, md, lg, xl, xxl

Unit III — Layout Components

Navbar & Navigation

Bootstrap's responsive navigation bar — the header of almost every website.

📚 Understanding Navbars

A navbar starts with .navbar and .navbar-expand-{breakpoint}. Key parts: .navbar-brand (logo), .navbar-toggler (hamburger), .navbar-nav + .nav-link (links).

Navigation elements: .nav (basic), .nav-tabs (tabbed), .nav-pills (pill-style).

💡 The toggler uses data-bs-toggle="collapse" pointing to the .navbar-collapse element's ID.
📖 Key Classes
  • .navbar — Base wrapper
  • .navbar-expand-lg — Expand at lg breakpoint
  • .navbar-brand — Site logo/name
  • .navbar-toggler — Hamburger button
  • .navbar-nav, .nav-link — Nav links
  • .nav-tabs, .nav-pills — Inline navs
  • .fixed-top, .sticky-top — Positioning

Dropdown Menus

Toggleable, contextual menus for displaying lists of links and actions.

📚 Understanding Dropdowns

A dropdown needs a trigger (button with data-bs-toggle="dropdown") and a menu (.dropdown-menu with .dropdown-item links). Types: standard, split button (.dropdown-toggle-split), dropup, dropend, dropstart.

💡 Dropdowns require the Bootstrap JavaScript bundle (including Popper.js).
📖 Key Classes
  • .dropdown — Wrapper
  • .dropdown-toggle — Trigger button
  • data-bs-toggle="dropdown" — Activates JS
  • .dropdown-menu — Menu container
  • .dropdown-item — Each item
  • .dropup, .dropend, .dropstart — Direction

Breadcrumbs & Pagination

Navigation aids that help users understand where they are.

Flexible content containers with headers, footers, and images.

Card parts: .card (wrapper), .card-body (main content), .card-header/.card-footer (optional), .card-title/.card-text (content), .card-img-top (image).

Modals

Dialog boxes and pop-up windows triggered by user interaction.

📚 Understanding Modals

A modal has a trigger (data-bs-toggle="modal" + data-bs-target="#id") and the modal element. Structure: .modal → .modal-dialog → .modal-content → .modal-header/.modal-body/.modal-footer.

Sizes: .modal-sm, .modal-lg, .modal-xl. Center: .modal-dialog-centered. Close: data-bs-dismiss="modal".

Unit V — JavaScript Plugins

Tooltips & Popovers

Small informational overlays on hover or click.

📚 Understanding Tooltips & Popovers

Tooltips: Add data-bs-toggle="tooltip" and title="text". Place with data-bs-placement="top|right|bottom|left". Must initialize with JavaScript.

Popovers: Like tooltips but with title + body. Use data-bs-toggle="popover" + data-bs-content="body". Also require JS initialization.

💡 These are the only Bootstrap components that need manual JS initialization. All others work via data attributes alone.

Unit V — JavaScript Plugins

Scrollspy, Alerts & More

Auto-highlight nav on scroll, dismissible alerts, and more plugins.

📚 Understanding Scrollspy & Alerts

Scrollspy: Add data-bs-spy="scroll" + data-bs-target="#navId" to auto-update nav links based on scroll position.

Alerts: .alert .alert-success etc. Make dismissible with .alert-dismissible + close button with data-bs-dismiss="alert".

Sticky positioning: .sticky-top replaces the old Affix plugin. Toggle buttons: data-bs-toggle="button".

Display & Float Utilities

Control visibility, display type, and floating behavior.

📚 Understanding Display & Float

Display: .d-none (hide), .d-block, .d-inline, .d-flex, .d-grid. Responsive: .d-none .d-md-block = hidden on small, visible on medium+.

Float: .float-start (left), .float-end (right), .clearfix (clear floats). In modern Bootstrap, prefer Flexbox over floats for layout.

Unit VI — Utilities

Utilities API & Resources

The Utility API and common utility patterns.

📚 Understanding Utilities API

Bootstrap generates utility classes like .mt-3, .text-center, .d-flex using a Sass-based Utility API. Key patterns:

  • Spacing: .m-* (margin), .p-* (padding). Sides: t/b/s/e/x/y. Values: 0-5, auto.
  • Sizing: .w-25/50/75/100, .h-*
  • Borders: .border, .border-primary, .rounded
  • Shadows: .shadow-sm, .shadow, .shadow-lg
  • Opacity: .opacity-25/50/75/100
💡 Pattern: .{property}{side}-{breakpoint}-{size}. Example: .mt-md-3 = margin-top, size 3, from md breakpoint.
